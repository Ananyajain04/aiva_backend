🔐 Authentication
POST /api/users/register Public
Register a new patient or caregiver account.
Request Body
"name":     "string"
"email":    "string (unique)"
"password": "string"
"role":     "patient" | "caregiver"
Response · 201
"_id":   "string"
"name":  "string"
"email": "string"
"role":  "string"
"token": "JWT — store this, used in all protected routes"
400 — User already exists 500 — Server error
POST /api/users/login Public
Login and receive a JWT token.
Request Body
"email":    "string"
"password": "string"
Response · 200
"_id":   "string"
"name":  "string"
"email": "string"
"role":  "string"
"token": "JWT"
401 — Invalid credentials 500 — Server error
GET /api/users/me Protected
Returns the profile of the currently logged-in user.
Response · 200
"_id":               "string"
"name":              "string"
"email":             "string"
"role":              "string"
"linkedCaregiverId": "ObjectId | null"  // patients only
401 — Unauthorized 404 — User not found
GET /api/users/my-patients Protected · Caregiver only
Returns all patients linked to the logged-in caregiver. Use the returned _id values as patientId / userId in subsequent requests.
Response · 200
[
  {
    "_id":               "string"  // use this as patientId
    "name":              "string"
    "email":             "string"
    "role":              "patient"
    "linkedCaregiverId": "ObjectId"
  }
]
403 — Only caregivers can access this 401 — Unauthorized
POST /api/users/link Protected · Caregiver only
Links a patient to the logged-in caregiver. The caregiver's ID is read from the token — do not pass it in the body.
Request Body
"patientId": "MongoDB ObjectId of the patient"
Response · 200
"message": "Patient linked successfully"
400 — Invalid patient 401 — Unauthorized
📊 Activity Logs
Logs are always created for the currently authenticated user — userId is taken from the JWT token, not the request body. Caregivers can read logs for their linked patients via GET /api/logs/:userId.
POST /api/logs Protected
Create a new activity log entry for the authenticated user.
Request Body
"activity": "string"           // required — e.g. "ate_food", "missed_lunch", "idle"
"source":   "app" | "sensor" | "caregiver" | "system"   // default: "app"
"type":     "activity" | "medication" | "reminder" | "sensor" | "behaviour"  // default: "activity"
"metadata": {}               // optional — any extra context (e.g. sensor readings, step times)
Response · 201
"_id":       "string"
"userId":    "string"
"activity":  "string"
"source":    "string"
"type":      "string"
"metadata":  {}
"createdAt": "date"
"updatedAt": "date"
400 — Invalid source or type 401 — Unauthorized
GET /api/logs/:userId Protected
Fetch logs for a patient. Patients can only fetch their own. Caregivers can only fetch their linked patient's logs.
Query Params (all optional)
type    — filter by type:   activity | medication | reminder | sensor | behaviour
source  — filter by source: app | sensor | caregiver | system
from    — start date: "2026-01-01"
to      — end date:   "2026-03-27"
Example
GET /api/logs/64abc123?type=medication&from=2026-01-01
Response · 200
[
  {
    "_id":       "string"
    "userId":    "string"
    "activity":  "string"
    "source":    "string"
    "type":      "string"
    "metadata":  {}
    "createdAt": "date"
  }
]  // sorted newest first
403 — Access denied 401 — Unauthorized
